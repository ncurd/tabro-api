# 对外接口文档

本文档描述当前项目对外暴露的主要 HTTP 接口。网关兼容接口使用 `Authorization: Bearer <API_KEY>`；控制台、用户、支付和管理接口使用登录态 JWT 或管理员鉴权。

> 说明：本文档先覆盖公开调用者最常用的接口面，详细字段以服务端实现和上游兼容协议为准。音频、视频接口已经接入现有账号体系，但真实供应商调用需要先配置 Azure、DashScope 和 Volcengine Ark 账号。

## 基础约定

| 项目 | 说明 |
| --- | --- |
| 网关 Base URL | `https://<host>` |
| 控制台 API Base URL | `https://<host>/api/v1` |
| 网关认证 | `Authorization: Bearer <API_KEY>` |
| 控制台认证 | `Authorization: Bearer <JWT>` |
| 请求格式 | JSON，媒体二进制响应除外 |
| 错误格式 | 网关接口尽量保持对应上游兼容格式；控制台接口使用项目统一响应结构 |

## 网关接口

网关接口用于把客户端请求路由到已配置的账号、分组和上游平台。分组平台决定调度范围；部分接口会根据请求模型进一步校验平台。

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/v1/messages` | Anthropic Messages 兼容入口 |
| `POST` | `/v1/messages/count_tokens` | Anthropic token 计数入口 |
| `POST` | `/v1/chat/completions` | OpenAI Chat Completions 兼容入口 |
| `POST` | `/v1/responses` | OpenAI Responses 兼容入口 |
| `GET` | `/v1/responses` | OpenAI Responses WebSocket 入口 |
| `POST` | `/v1/embeddings` | Embeddings 兼容入口 |
| `POST` | `/v1/images/generations` | OpenAI 图像生成入口 |
| `GET` | `/v1/models` | 模型列表 |

常用请求头：

```http
Authorization: Bearer sk-your-api-key
Content-Type: application/json
```

### Chat Completions 示例

```bash
curl https://<host>/v1/chat/completions \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.1",
    "messages": [
      { "role": "user", "content": "Hello" }
    ]
  }'
```

### Responses 示例

```bash
curl https://<host>/v1/responses \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.1",
    "input": "写一个简短的产品介绍"
  }'
```

## 音频生成

音频生成使用 Azure Speech Text-to-speech，并复用现有账号和分组体系。账号平台应配置为 `azure_speech`。

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/v1/audio/speech` | 实时 TTS，直接返回音频二进制 |
| `POST` | `/v1/audio/speech/jobs` | 长文本/批量 TTS，创建异步任务 |
| `GET` | `/v1/audio/speech/jobs/:id` | 查询异步任务状态和结果 |

账号凭证建议：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `subscription_key` | 是 | Azure Speech 资源密钥 |
| `region` | 是 | Azure 区域，例如 `eastus` |
| `tts_endpoint` | 否 | 自定义实时 TTS Endpoint |
| `batch_endpoint` | 否 | 自定义 Batch synthesis Endpoint |

实时 TTS 请求示例：

```bash
curl https://<host>/v1/audio/speech \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  --output speech.mp3 \
  -d '{
    "model": "azure-tts",
    "input": "欢迎使用 Tabro 音频生成接口。",
    "voice": "zh-CN-XiaoxiaoNeural",
    "language": "zh-CN",
    "response_format": "mp3",
    "speed": 1
  }'
```

异步任务请求示例：

```bash
curl https://<host>/v1/audio/speech/jobs \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "azure-tts",
    "input": "这里可以放较长的旁白、课程或播客文案。",
    "voice": "zh-CN-XiaoxiaoNeural",
    "language": "zh-CN",
    "response_format": "mp3"
  }'
```

任务响应会包含 `id`、`status`、`provider`、`upstream_task_id` 等字段。随后使用：

```bash
curl https://<host>/v1/audio/speech/jobs/audjob_xxx \
  -H "Authorization: Bearer sk-your-api-key"
```

## 视频生成

视频生成目前支持 DashScope HappyHorse 和 Volcengine Ark Seedance 2.0。两者均为异步任务。

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/v1/videos/generations` | 创建视频生成任务 |
| `GET` | `/v1/videos/generations/:id` | 查询任务状态和视频结果 |

模型到平台的默认匹配：

| 模型关键字 | 平台 | 账号平台 |
| --- | --- | --- |
| `happyhorse` | 阿里云 DashScope | `dashscope` |
| `seedance` | 火山方舟 Ark | `volcengine_ark` |

### DashScope HappyHorse 示例

```bash
curl https://<host>/v1/videos/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "happyhorse-1.0-r2v",
    "prompt": "[Image 1]中的人物拿起[Image 2]中的折扇，镜头从中景推进到面部特写。",
    "media": [
      {
        "type": "reference_image",
        "url": "https://example.com/person.jpg"
      },
      {
        "type": "reference_image",
        "url": "https://example.com/fan.jpg"
      }
    ],
    "resolution": "720P",
    "ratio": "16:9",
    "duration": 5,
    "watermark": false
  }'
```

DashScope 账号凭证：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `api_key` | 是 | DashScope API Key |
| `base_url` | 否 | 地域 Endpoint，例如 `https://dashscope.aliyuncs.com`、`https://dashscope-intl.aliyuncs.com` |

### Seedance 2.0 示例

```bash
curl https://<host>/v1/videos/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "doubao-seedance-2-0-260128",
    "prompt": "参考图片1中的产品外观，生成一段干净的产品展示视频。",
    "media": [
      {
        "type": "reference_image",
        "url": "https://example.com/product.png"
      },
      {
        "type": "reference_audio",
        "url": "https://example.com/music.mp3"
      }
    ],
    "ratio": "16:9",
    "duration": 8,
    "generate_audio": true,
    "watermark": true
  }'
```

Seedance 账号凭证：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `api_key` | 是 | Volcengine Ark API Key |
| `base_url` | 否 | 默认 `https://ark.cn-beijing.volces.com/api/v3` |

查询视频任务：

```bash
curl https://<host>/v1/videos/generations/vidjob_xxx \
  -H "Authorization: Bearer sk-your-api-key"
```

成功后响应中的 `result_url` 为上游返回的视频地址。部分上游临时地址有效期较短，应尽快下载或转存。

## 任务状态

音频和视频异步任务共用一套状态语义。

| 状态 | 含义 |
| --- | --- |
| `queued` | 本地已创建，等待上游处理 |
| `running` | 上游处理中 |
| `succeeded` | 成功，可读取结果 URL |
| `failed` | 上游或本地处理失败 |
| `canceled` | 任务取消 |
| `unknown` | 上游任务不存在或已过期 |

## 常见错误

| HTTP 状态 | 常见原因 | 建议处理 |
| --- | --- | --- |
| `400` | 请求字段缺失、模型与平台不匹配 | 检查 `model`、`media`、账号平台 |
| `401` | 缺少或错误的 API Key/JWT | 重新生成或刷新凭证 |
| `403` | 用户无权限访问资源 | 检查分组、角色、任务归属 |
| `429` | 触发限流或上游额度不足 | 降低请求频率或切换账号 |
| `5xx` | 上游失败或无可用账号 | 检查账号状态、代理、上游服务状态 |

## 配置检查清单

上线前建议至少完成以下检查：

1. API Key 已绑定到正确分组。
2. 分组平台与账号平台一致。
3. 模型名已在对应账号或渠道允许范围内。
4. 音频账号配置了 Azure `subscription_key` 和 `region`。
5. DashScope 视频账号配置了同地域 `api_key` 和 `base_url`。
6. Seedance 视频账号配置了 Ark `api_key`，并已开通对应模型。
7. 异步任务查询接口只允许任务创建者读取。
